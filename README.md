# Git Flow

Этот документ фиксирует правила ветвления и Pull Request’ов в проекте.

---

## Основные ветки

### `main`
- Содержит только **стабильные** версии кода, готовые к релизу/демо.

### `develop`
- Главная ветка для **активной разработки**.
- Все новые функции и исправления собираются в `develop`.
- Отражает состояние разработки перед тем, как изменения будут включены в релиз.

### `feature/*`
- Ветки для разработки новой функциональности.
- **Создаются от `develop`** и после завершения **сливаются обратно в `develop` через Pull Request**.
- Формат имени ветки: `nickname/название-фичи`.

**Примеры**
- `nastya/inventory-ui`
- `vanya/player-movement`

---

## Структура работы

1) **Разработка ведётся в `develop`.**

2) **Каждая фича разрабатывается в отдельной `feature`-ветке**, созданной от `develop`.

3) **После завершения фичи создаётся Pull Request на @Vanadium19.**

4) **Мердж в `develop` технически возможен только после апрува PR.**

5) **Мерджим фичу в `develop` через Squash (Git squash / Squash and merge).**

6) **После мерджа в `develop` feature-ветка удаляется.**

---

## Конфликты при мердже

**PS:** В случае конфликтов при мердже — **все конфликты решаются в feature-ветке** (не в `develop`).
